# EEXPR-57: Fix syncI9InfoToPEoContract updating wrong contract during entity transfers

## Problem Statement

During entity transfers, the `syncI9InfoToPEoContract` function in `peo_contract_service.ts` syncs I-9 data to the **wrong contract**. This causes entity transfers for Z04 (Alien Authorized) employees to fail with validation errors like "i9AlienRegistrationExpiration is required" even when the data exists in the database.

### Root Cause

The bug occurs in the following call chain:

1. `CopyI9DataStep.execute()` copies I9Section1/I9Section2 data from old to new contract
2. It calls `syncI9InfoToPEoContract(newContractOid)` with the correct new contract OID
3. `syncI9InfoToPEoContract` retrieves the contract by OID and gets the profileId
4. It then calls `updateDefaultPEOContractWithI9(profileId, {...})` passing **only the profileId**
5. `updateDefaultPEOContractWithI9` calls `getDefaultContract(profileId)`
6. **BUG:** `getDefaultContract` returns the **first active contract** for the profile (non-deterministic order)
7. During entity transfers, both old and new contracts are active, so it may return the **old contract**
8. The I-9 data gets synced to the wrong (old) contract

### Why Z04 Employees Are Affected Visibly

- Z04 (Alien Authorized) employees have stricter I-9 validation: paired fields like `alienRegNumber` + `expiration`
- Other citizenship statuses (Z01, Z03) have either no I-9 requirements or only `alienRegNumber` required
- The bug affects ALL transfers, but only Z04 employees hit validation failures due to missing synced data

## Technical Context

### Affected Files

- **backend/services/peo/peo_contract_service.ts**
  - `getDefaultContract()` (lines 3923-3936): Returns first active contract
  - `updateDefaultPEOContractWithI9()` (lines 3984-4016): Needs contractId param
  - `syncI9InfoToPEoContract()` (lines 4018-4031): Has contractOid but passes profileId

### Code Flow

```
CopyI9DataStep.execute()
    │
    ▼
syncI9InfoToPEoContract(newContractOid)  ← Has correct OID
    │
    ├─ db.peo_contract.findOne({ where: { deelContractOid } })  ← Finds correct contract
    │
    ▼
updateDefaultPEOContractWithI9(profileId, i9Data)  ← BUG: Loses contract reference
    │
    ▼
getDefaultContract(profileId)  ← Returns WRONG contract (first active, non-deterministic)
```

### Proposed Fix

```typescript
// peo_contract_service.ts

async updateDefaultPEOContractWithI9(
    profileId: number,
    i9Data: {...},
    contractId?: number  // NEW: Optional parameter to target specific contract
): Promise<void> {
    let contract;
    if (contractId) {
        contract = await this.db.peo_contract.findByPk(contractId);
    } else {
        contract = await this.getDefaultContract(profileId);  // Fallback for existing callers
    }
    // ... rest of function
}

async syncI9InfoToPEoContract(contractOid: string): Promise<void> {
    const contract = await this.db.peo_contract.findOne({
        where: { deelContractOid: contractOid }
    });
    // Pass contract.id to ensure correct contract is updated
    await this.updateDefaultPEOContractWithI9(
        contract.profileId,
        { ...i9Data },
        contract.id  // NEW: Pass contract ID
    );
}
```
