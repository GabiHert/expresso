# Plan: Collapse All Work Items Functionality

## Understanding

The user wants to collapse all work items in the tree view, similar to how sessions and files changed sections can be collapsed.

## Current Implementation

### Tree Hierarchy
```
SectionItem (in_progress|todo|done) [Expanded]
  └─ TaskItem [Collapsed]
      ├─ SessionsSection [Collapsed]     ← Can be collapsed
      │   └─ SessionItem[]
      ├─ WorkItemNode[]                  ← Currently LEAF NODES (no collapse)
      └─ FilesChangedSection [Collapsed] ← Can be collapsed
          └─ ShadowFileItem[]
```

### Key Finding
Work items are currently rendered as **leaf nodes** with `TreeItemCollapsibleState.None`. This means:
- They cannot be collapsed/expanded
- They appear as direct children of TaskItem
- There's no "WorkItemsSection" wrapper like SessionsSection or FilesChangedSection

### How Sessions/Files Work
Both sessions and files changed have a **section wrapper**:
- `SessionsSection` (line 579-603 in TaskTreeProvider.ts)
- `FilesChangedSection` (line 505-517 in TaskTreeProvider.ts)

These sections are collapsible containers that group their child items.

## Questions

**Need clarification on user intent:**
1. Do you want a "Work Items" section that wraps all work items and can be collapsed as a group?
2. Or do you want individual work items to be expandable (with sub-items inside)?

Option 1 seems most consistent with existing patterns (SessionsSection, FilesChangedSection).
