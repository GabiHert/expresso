# ENGESC-23600: Tech Ops Endpoint for Missing PEO Employments

## Problem Statement

Entity transfer contracts are missing employment records in the Employment microservice. This is caused by a bug where the `afterCommit` callback in `peoContractService.createContract()` fails silently because `jobTitle` is not passed from `CreateContractStep`.

**Root Cause**:
- Entity transfer only passes `jobCode` in `peopleFirstPayload.validatedData`
- The Employment microservice requires `jobTitle` (Zod validation)
- Error is swallowed as unhandled promise rejection

**Affected Contracts**: 19 contracts for Amae Health, Inc. (org 386153) transferred on 2026-01-01 and 2026-01-03.

**Related Escalations**: ENGESC-23650, ENGESC-23600

## Acceptance Criteria

- [ ] New tech_ops endpoint `POST /admin/peo/tech_ops/contract/:oid/create_missing_employment` exists
- [ ] Endpoint requires `startDate` parameter (YYYY-MM-DD)
- [ ] Endpoint checks if employment already exists (idempotent - skips if exists)
- [ ] Endpoint fetches all data from NEW contract (no old contract needed)
- [ ] Endpoint resolves `jobTitle` from `jobCode` (falls back to jobCode if not found)
- [ ] Endpoint follows existing tech_ops patterns (Joi validation, permissions, error handling)
- [ ] Endpoint is documented with wiki reference comment
- [ ] Can be used to fix the 19 affected contracts

### Required Data for Employment Creation

All data is fetched from the NEW contract - no old contract reference needed:

```typescript
{
  contractOid: string,           // NEW contract.oid
  country: "US",                 // NEW contract.country
  state: string,                 // NEW peo_contracts.state_code
  clientTeamId: number,          // NEW contract.team_id
  jobTitle: string,              // Resolved from NEW peo_contracts.job_code
  rate: string,                  // NEW peo_contracts.pay_rate
  scale: "YEAR" | "HOUR",        // NEW peo_contracts.pay_method

}
```

## Implementation Approach

1. **Service Method First**: Add `createMissingEmploymentForContract()` to `peoContractService`
2. **Endpoint Second**: Add POST endpoint to `tech_ops.ts` that calls the service
3. **Test with one contract**: Verify it works for one affected contract
4. **Fix all 19**: Run against all affected contracts

## Risks & Considerations

1. **Idempotency**: Must check if employment exists before creating (avoid duplicates)
2. **jobTitle Resolution**: If jobCode doesn't exist in job master, should fallback to using jobCode as jobTitle
3. **Transaction Safety**: Employment creation is an HTTP call - not transactional with other DB operations
4. **Permissions**: Requires `admin:contracts.write` permission

## Testing Strategy

1. **Unit Test**: Mock employment service and verify correct data is passed
2. **Manual Test**:
   - Call endpoint for a contract that already has employment (should skip)
   - Call endpoint for affected contract (should create)
   - Verify employment exists in Employment microservice
