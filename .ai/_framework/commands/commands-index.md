# AI Task Framework

A reusable, three-layer system for AI-assisted development workflows.

## Quick Start

1. Copy the entire `.ai/` directory to your project
2. Run `/init` to bootstrap (generates manifest, context, etc.)
3. Use `/task-create` to start working

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                        DOMAIN LAYER                             │
│  .ai/docs/ - Accumulated project knowledge                      │
├─────────────────────────────────────────────────────────────────┤
│                       PROJECT LAYER                             │
│  .ai/_project/ - Project configuration (manifest.md)           │
├─────────────────────────────────────────────────────────────────┤
│                      FRAMEWORK LAYER                            │
│  .ai/_framework/ - Generic commands and templates               │
└─────────────────────────────────────────────────────────────────┘
```

## Commands

| Command | Description |
|---------|-------------|
| `/init` | Bootstrap a new project with the framework |
| `/enhance` | Add new repos, conventions, or documentation |
| `/task-create` | Create a new development task |
| `/task-start` | Begin working on a task |
| `/task-resume` | Resume an in-progress task (session continuity) |
| `/task-work` | Implement work items |
| `/task-done` | Complete and log a task |
| `/task-status` | View task dashboard |
| `/task-review` | Run code review |
| `/task-explore` | Explore codebase context |
| `/document` | Create documentation |
| `/help` | Show available commands |
| `/task-orchestrate` | Orchestrate task with background agents and worktree isolation |
| `/command-create` | Create new framework command |
| `/command-extend` | Extend commands at project level |

## Templates

| Template | Purpose |
|----------|---------|
| `context.md` | Project entry point (generated by /init) |
| `doc.md` | Standard documentation structure |
| `task-readme.md` | Task note with frontmatter (type, status, summary) |
| `work-item.md` | Individual work item note with frontmatter status |
| `repo-readme.md` | Per-repo documentation root |

## Key Concepts

### Manifest

The `.ai/_project/manifest.md` is the single source of truth for:
- Project identity
- Repository list
- Available MCPs
- Conventions (commits, branches, JIRA)
- Agent preferences

Data is stored in the note's frontmatter and read via `get_frontmatter("_project/manifest.md")`.

### Anti-Lost Protocol

Every file includes headers that remind the AI:
- Which layer it's in
- What to read before proceeding
- How to navigate

### Task Lifecycle

```
todo → in_progress → done
```

Task status is stored in each task note's frontmatter (`status` field). To find tasks by status, use `search_notes("type: task status: in_progress")` or scan `tasks/` and check frontmatter.

Work items live flat inside the task folder, each with their own frontmatter `status`:

```
.ai/tasks/TASK-ID/
├── TASK-ID.md          ← frontmatter: type, status, summary counts
├── TASK-ID-01.md       ← work item, frontmatter: status: todo|in_progress|done
├── TASK-ID-02.md
└── TASK-ID-03.md
```

Advancing a task or work item means updating its `status` frontmatter field — no file moves required.
